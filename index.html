Segurança — Boas Práticas
  • No Supabase, ative Row Level Security (RLS) em todas as tabelas
  • Configure políticas para que a tabela pedidos aceite INSERT de anon, mas SELECT/UPDATE apenas de usuários autenticados
  • A tabela produtos deve ter apenas SELECT permitido para anon — inserts só via service role
  • Nunca exponha a Service Role Key no HTML — use apenas a Anon/Publishable Key
  • No GitHub, certifique-se de que o repositório é privado e nunca commite credenciais hardcoded
  • Adicione o domínio do site nas Allowed Origins do Supabase (Authentication → URL Configuration)
